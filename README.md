# Utility

Remote administration tool — a web dashboard for managing Metasploit Framework sessions, payloads, listeners, and modules.

## Architecture

```
Caddy (:80/:443) → Next.js Dashboard (:3000) → MSF RPC (TCP :55553) → PostgreSQL (:5432)
```

## Quick Start

```bash
# 1. Clone and enter
git clone https://github.com/balestar/utility.git
cd utility

# 2. Start the full stack
docker compose up -d

# 3. Access the dashboard
open http://localhost:3000
```

## Features

| Module | Description |
|---|---|
| **Dashboard** | Overview with stat cards and quick actions |
| **Payloads** | Generate backdoors (exe, elf, python, powershell, etc.) |
| **Listeners** | Start/stop multi/handler listeners |
| **Sessions** | View active shells and meterpreter sessions |
| **Modules** | Browse exploits, payloads, and auxiliary modules |
| **Workspaces** | Manage isolated engagement environments |
| **Settings** | App lock, connection info, quick commands |

## Factory Reset Survival

A factory reset wipes the entire disk, but recovery is **one command away**.

### Recovery command (on clean macOS)

```bash
curl -fsSL https://raw.githubusercontent.com/balestar/utility/main/scripts/factory-recovery.sh | sh
```

This single command:

1. Installs Xcode Command Line Tools (if missing)
2. Installs Homebrew (if missing)
3. Installs Docker Desktop (if missing)
4. Clones the repository
5. Creates a secure `.env` with auto-generated secrets
6. Builds and starts the full Docker stack
7. Installs a LaunchAgent for auto-start on future boots
8. Prints all access credentials

**With data restoration from cloud:**

```bash
curl -fsSL https://raw.githubusercontent.com/balestar/utility/main/scripts/factory-recovery.sh | \
  DATA_URL=https://my-storage.example.com/utility-latest.tar.gz sh
```

The backup tarball should contain:
- `payloads/` — generated payload files
- `env` — environment configuration
- `msf-db.sql` — exported PostgreSQL dump
- `msf-container-payloads/` — payloads from inside the MSF container

### Backup regularly

```bash
# Local backup
./scripts/backup.sh

# Cloud backup (S3)
./scripts/backup.sh s3://utility-backups/

# Cloud backup (Cloudflare R2)
R2_ENDPOINT_URL=https://<account>.r2.cloudflarestorage.com \
  ./scripts/backup.sh r2://utility-backups/

# Restore from backup
./scripts/restore.sh ./backups/utility-backup-20260620.tar.gz
./scripts/restore.sh s3://utility-backups/utility-backup-20260620.tar.gz
```

### What survives a factory reset

| Item | Survives? | How |
|---|---|---|
| **Source code** | ✓ | Hosted on GitHub. One-command clone & build. |
| **Docker images** | ✓ | Pulled from registries on `docker compose up`. |
| **Environment config** | ✓ | Auto-generated by recovery script, or restored from backup. |
| **Auto-start config** | ✓ | Installed by recovery script (LaunchAgent/systemd). |
| **Generated payloads** | ✓ | Restored from cloud backup. |
| **MSF database (sessions, workspaces)** | ✓ | Restored from cloud backup. |
| **Live sessions** | ✗ | Lost on reset — target machines would need new beacons. |
| **Docker volumes** | ✗ | Wiped with disk — DB data restorable from backup. |
| **PIN lock hash** | ✗ | Stored in localStorage — set a new one. |

## Security

- PIN lock screen with auto-lock after inactivity
- Panic button (double-click bottom-right corner to lock)
- API key authentication on all routes
- Caddy reverse proxy with basic auth + TLS
- Generic "Utility" branding (PWA named "Utility")

## Scripts

| Script | Purpose |
|---|---|
| `scripts/factory-recovery.sh` | Full recovery after factory reset |
| `scripts/start-after-reboot.sh` | Start stack after reboot |
| `scripts/backup.sh` | Backup payloads, config, and DB |
| `scripts/restore.sh` | Restore from a backup archive |
| `scripts/set-caddy-password.sh` | Generate a new Caddy basic auth hash |
